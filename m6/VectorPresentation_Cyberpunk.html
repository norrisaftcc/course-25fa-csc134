The Problem With Arrays

const int MAX_PARTY = 4; // What if we want 5? string partyNames[MAX_PARTY]; // If we need more space... we're stuck
  • Fixed size at compile time
  • No bounds checking (crashes!)
  • Can't easily grow or shrink
  • Manual size tracking everywhere
  • Messy function interfaces

You've been fighting these limitations. Time for an upgrade.

Enter: std::vector

#include <vector> #include <string> vector<string> partyNames; // Starts empty, grows as needed!

Vector Superpowers:

  • Dynamic size - grows and shrinks automatically
  • Knows its own size (no separate SIZE variable)
  • Bounds checking available (.at() method)
  • Easy to add/remove elements
  • Works with STL algorithms

Cool Thing #1: Dynamic Growth

void addItem(const string& item) { inventory.push_back(item); // Vector grows automatically! cout << "Added: " << item << endl; } void removeItem(const string& item) { auto it = find(inventory.begin(), inventory.end(), item); if (it != inventory.end()) { inventory.erase(it); // Vector shrinks automatically! } }

Try this with an array: Manual shifting, tracking "used" size, managing empty spaces...

Vectors handle all of this automatically.

Cool Thing #2: Size Tracking

void displayInventory() { cout << "\n=== INVENTORY (" << inventory.size() << " items) ===" << endl; if (inventory.empty()) { // Handy method! cout << "Empty inventory." << endl; return; } for (size_t i = 0; i < inventory.size(); i++) { cout << i + 1 << ". " << inventory[i] << endl; } }

Cool Thing #3: STL Algorithms

This is where vectors truly shine. Modern C++ at its best.

#include <algorithm> void sortInventory() { sort(inventory.begin(), inventory.end()); // One line to sort! Try doing this with an array... } bool hasItem(const string& item) { // Search without writing a loop return find(inventory.begin(), inventory.end(), item) != inventory.end(); } int countPotions() { // Count items matching a condition with lambdas! return count_if(inventory.begin(), inventory.end(), [](const string& item) { return item.find("Potion") != string::npos; }); }

Vector vs Array: Head-to-Head

Feature Array Vector
Size Fixed at compile time Dynamic
Size Tracking Manual Automatic (.size())
Bounds Check None Available (.at())
Add/Remove Manual shifting push_back(), erase()
STL Algorithms Limited Full support
Function Passing Array + size Just vector

The Global Vector Pattern

#include <vector> #include <string> using namespace std; // Global inventory - multiple functions can access it vector<string> inventory; void addItem(const string& item); void removeItem(const string& item); void displayInventory(); void sortInventory();

Why This Works for Learning:

  • Multiple functions naturally share game state
  • See how different operations modify the vector
  • Mirrors how game systems actually work
  • Builds toward understanding parameter passing

Note: In production code, we'd use classes and encapsulation. But for learning functions with shared data, global vectors are a great teaching tool.

Key Takeaways

  • Vectors are arrays with superpowers
  • They manage their own size and memory
  • They integrate with STL algorithms
  • They're safer and more convenient
  • They're the standard choice in modern C++
